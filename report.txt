## 1. Introduction
The Medicine Reminder App is a comprehensive desktop application built with Python and Tkinter, designed to help users manage their medication schedules. It features a robust notification system, multi-user support, and a clean, intuitive interface. The app is engineered to be cross-platform and easily convertible to a mobile application using Kivy.

## 2. File Structure
```
medicine-reminder-app/
├── main.py                 # Main application with UI components
├── database.py            # Database operations and management
├── notification_manager.py # Notification handling system
├── auth_manager.py        # Authentication and user management
├── models.py             # Data models and classes
├── requirements.txt      # Python dependencies
└── README.md            # Project documentation
```

### Purpose of Each File:
- `main.py`: Core application file containing UI components and application logic
- `database.py`: Handles all database operations including CRUD operations for reminders and users
- `notification_manager.py`: Manages cross-platform notifications using Plyer
- `auth_manager.py`: Handles user authentication, login, and session management
- `models.py`: Contains data models for Users, Reminders, and other entities
- `requirements.txt`: Lists all Python package dependencies
- `README.md`: Project documentation and setup instructions

## 3. Core Components

### A. Database Manager (database.py)
- Uses SQLite for persistent storage
- Implements field-level encryption for sensitive data
- Handles all CRUD operations for:
  - Users and family members
  - Medicine reminders
  - Intake logs
- Features automatic schema migrations
- Implements data validation and integrity checks

### B. Authentication Manager (auth_manager.py)
- Manages user sessions
- Handles login/logout operations
- Implements password hashing using bcrypt
- Provides user profile management
- Supports multiple user types (self/family)

### C. Notification Manager (notification_manager.py)
- Cross-platform notification system using Plyer
- Background thread for checking due reminders
- Supports different notification types:
  - Reminder notifications
  - Test notifications
  - Immediate notifications
- Implements IST timezone handling

## 4. Database Design

### Tables:

#### Users Table
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    username TEXT UNIQUE,
    password_hash TEXT,
    is_self BOOLEAN DEFAULT 0,
    owner_id INTEGER,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_profile BOOLEAN DEFAULT 0
)
```

#### Reminders Table
```sql
CREATE TABLE reminders (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    medicine_name TEXT NOT NULL,
    dose TEXT NOT NULL,
    time_slot TEXT NOT NULL,
    specific_time TEXT NOT NULL,
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    take_with_food BOOLEAN DEFAULT 1,
    medicine_image_path TEXT,
    is_taken BOOLEAN DEFAULT 0,
    taken_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    days_of_week TEXT,
    FOREIGN KEY (user_id) REFERENCES users (id)
)
```

#### Intake Logs Table
```sql
CREATE TABLE intake_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    reminder_id INTEGER NOT NULL,
    user_id INTEGER NOT NULL,
    intake_date DATE NOT NULL,
    status TEXT NOT NULL,
    taken_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (reminder_id) REFERENCES reminders (id),
    FOREIGN KEY (user_id) REFERENCES users (id)
)
```
